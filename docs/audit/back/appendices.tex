\section*{B. Metodologia ocen}
\addcontentsline{toc}{section}{B. Metodologia ocen}

Oceny poszczególnych obszarów (skala 0--10) zostały ustalone na podstawie następujących kryteriów:

\begin{description}
  \item[9--10] Wzorcowa implementacja, brak istotnych problemów, gotowa do produkcji bez zmian.
  \item[7--8] Solidna implementacja z~drobnymi problemami. Gotowa do produkcji po minor fixes.
  \item[5--6] Funkcjonalna implementacja z~istotnymi problemami. Wymaga pracy przed produkcją.
  \item[3--4] Poważne problemy strukturalne lub koncepcyjne. Wymaga redesignu.
  \item[0--2] Niefunkcjonalna lub brak implementacji.
\end{description}

Wagi per obszar w~ocenie ogólnej:
\begin{itemize}
  \item Jakość kodu: 15\%
  \item Parsery: 10\%
  \item Pipeline AI: 15\%
  \item Interfejs: 10\%
  \item Schematy/walidacja: 10\%
  \item Obsługa błędów: 10\%
  \item Walidacja psychologiczna: 15\%
  \item Bezpieczeństwo: 10\%
  \item Rozrywka vs nauka: 5\%
\end{itemize}
